Hey folks,
In case you were planning to use the go implementation for your upcoming
participation, be aware of this issue with go 1.9.4 --
https://github.com/golang/go/issues/23739

The issue did come up for me on a debian-based system when trying to
execute the go code from https://github.com/FiloSottile/powersoftau.

I ran into that problem and (as suggested in the github issue for a
workaround) downgrading to go 1.9.3 solved the problem.

Just thought I'd save you some hassle in case you are hitting the same
error.
